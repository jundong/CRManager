Using Axon : Details for working on the Traffic Player page
Details for working on the Traffic Player page
To begin to edit the test settings, we click the Tracks tab.
Configure the Tracks in the Playlist for this test
A group of tracks makes up the Playlist for this test.
Each Track is a type of traffic that Axon sends to the network (the number before each track name indicates the OSI Layer for the encapsulation type or protocol of the track). Tracks can be traffic with a single packet size, a mix of packet sizes, or a mix of protocols. For example a track can be HTTP traffic or data in 64Byte frames.
You can select from predefined tracks for common cases or from tracks that you had designed and want to use again (Favorites).
a
Click the Playlists tab.
b
Open the playlist for editing by clicking the arrow. Now you can add a playlist by dropping from the Playlists tab.
c
You have the option to add tracks to the playlist by dropping from the Tracks tab into the editing area at the bottom of the Playlist pad.
In the example, we added the End‑to‑End playlist. Within a track, differing data encapsulation types (or protocols) are interspersed randomly in the traffic. The size of the bar in the Playlist pad represents the percentage of the data that is made of the particular encapsulation type or protocol.
d
Click the disk to save the playlist definition (for reuse in a different test, for example).
Note You cannot mix L2/L3 items with L4-L7 items within a playlist. You can, however, configure and run any mix of traffic types in a test. To send mixed traffic, add another Traffic Player to the test (as we will describe in a moment).
Configure the Endpoints
 
Tx ports appear on the left and Rx ports on the right.
a
Click the down‑arrow to select a device for editing.
b
Click any port to open its editing pad.
c
Click the Endpoints tab. You can select from predefined endpoints for common cases or from endpoints that you had designed and want to use again (Favorites). Add an endpoint configuration by dropping from the Endpoints tab into the editing area below the ports.
d
Click the disk to save the endpoint definition (for reuse in a different test, for example).
Name
Specify a meaningful name for the endpoint that represents the configuration settings that you made.
Favorite
Check the box to add the endpoint to the list of Favorites that appear whenever endpoints are listed.
VLAN ID
Specify the VLAN tag to apply to both endpoints and traffic sent from the endpoints.
The default value of None implies that no VLAN tag is present.
DHCP
If either of the DHCP options are enabled (IPv4 or IPv6), then the endpoints will attempt to retrieve all IP address information from DHCP or DHCPv6 servers.
Note 
If no DHCP servers are available on the test network, then the test will fail to run.
Default: No DHCP
Starting IP
Specify the beginning IP address to use for test endpoints. The address can be either IPv4 or IPv6.
Ending IP
Specify the ending IP address (IPv4 or IPv6) to use for test endpoints. The address must match the version of Starting IP. All IP addresses between Starting IP. and Ending IP are used for test endpoints
If Ending IP is blank, then only Starting IP is used for an endpoint.
Prefix
Specify the network prefix length to use for test endpoints. The value determines which addresses are on the local network and which addresses must be reached via the address specified by Gateway.
Gateway
Specify the IP address (IPv4 or IPv6) of the router to use when sending packets to endpoints on non-local networks. The IP address must match the version of Starting IP..
Tags
To enable you to quickly access appropriate endpoints when you take Axon on the road, you might want to organize port configurations by customer, by location, or by a tag that you specify. Specify a tag by typing into the text box.
Configure the Traffic
Specify the traffic rate in Mbps by typing it into the text box.
Traffic Player
Together, the tracks, playlists, endpoints, and traffic rate make up a traffic player The example is currently named “Traffic Player 1”.
•
You can add any number of players to a test — one player might generate L2/L3 traffic and another player might generate L4-L7 traffic. Click Add Another Traffic Player
•
For any test, players run concurrently
•
When you save a test, you save all settings in all players
